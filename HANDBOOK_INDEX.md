# Handbook Index

This handbook captures the foundational principles, identity, brand positioning, and decision-making framework used throughout the repository.

## Foundation Dependency Chain

Engineering Philosophy -> Who I Am -> Brand -> Decision Framework

## Foundation Documents

1. `docs/00_FOUNDATION/01_ENGINEERING_PHILOSOPHY.md`
   - Defines the engineering principles that guide architecture, implementation, reliability, simplicity, documentation, and responsible AI.

2. `docs/00_FOUNDATION/02_WHO_I_AM.md`
   - Defines the identity, motivations, leadership posture, and long-term engineering perspective behind the repository.

3. `docs/00_FOUNDATION/03_BRAND.md`
   - Defines public positioning, voice, audience, brand pillars, and how the portfolio should communicate trust.

4. `docs/00_FOUNDATION/04_DECISION_FRAMEWORK.md`
   - Defines the canonical architectural and product decision-making framework for future documents and application development.
   - Version: v0.1
   - Status: Foundational Draft

## Usage Rule

Future documents should reference `docs/00_FOUNDATION/04_DECISION_FRAMEWORK.md` for decision logic instead of redefining architectural or product decision-making from scratch.

---

## Living Document

Engineering evolves continuously.

This document captures current principles rather than immutable truths.

When new evidence contradicts an existing belief, the document should be updated accordingly.

Strong engineering values learning over consistency.
