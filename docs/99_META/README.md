# AI Architect Handbook Core

This folder contains the meta documents that govern how the AI Architect Handbook is organised, written, versioned, and evolved.

These documents are the operating system of the repository.

## Core Documents

- `README.md` - welcome to the handbook core
- `HANDBOOK_INDEX.md` - knowledge graph and reading order
- `DOCUMENT_CONVENTIONS.md` - how documentation is written
- `GLOSSARY.md` - canonical terminology
- `REPOSITORY_PRINCIPLES.md` - rules that govern the repository
- `VERSIONING.md` - document versioning and status conventions
- `CHANGELOG.md` - meaningful handbook changes

## Usage

Start with the root `README.md`, then read `HANDBOOK_INDEX.md`.

Use `DOCUMENT_CONVENTIONS.md` before creating or renaming documents.

Use `REPOSITORY_PRINCIPLES.md` before making structural repository changes.

## Living Document

Engineering evolves continuously.

This document captures current principles rather than immutable truths.

When new evidence contradicts an existing belief, the document should be updated accordingly.

Strong engineering values learning over consistency.
